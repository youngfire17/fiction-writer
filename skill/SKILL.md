# Fiction Writer Skill

This skill orchestrates an autonomous fiction-book pipeline: premise → outline → draft → revise → export. It uses 8 specialized subagents and a knowledge library of craft references, anti-AI voice rules, genre conventions, and author-voice profiles.

## When to use

Invoke this skill when the user wants to:

- Start a new book or series from a premise (`/fiction-writer new "premise..."` or just describe the premise)
- Outline an existing project
- Draft the next chapter
- Run a continuity audit
- Revise a draft toward a target word count
- Export to manuscript or PDF

If the request is just "write me a paragraph in the style of X," this is overkill — answer directly.

## Repo layout

The skill assumes this repo structure (paths relative to repo root):

```
library/             craft references — load on demand by subagents
  anti-ai/           tells catalog + scrub rules (load on every draft)
  story-structure/   beat frameworks
  character/         creation, arcs, voice
  prose/             sentence-level craft
  world-building/    settings, magic, cultures
  genre/             per-genre conventions
  voice/             author profiles
subagents/           the 8 agent definitions
templates/           book outline, series bible, character sheet, chapter brief
tools/               build_continuity_index.py and other utilities
books/<slug>/        one folder per project
```

## The 8 subagents

Run in this pipeline order. See `subagents/<name>.md` for full role specs.

1. **world-builder** — designs setting + magic/tech + cultures + history. Writes `series-bible/world/`.
2. **character-designer** — creates the cast with full sheets. Writes `series-bible/characters/`.
3. **plotter** — produces beat sheet, chapter list, and setup→payoff graph. Writes `plot/`.
4. **drafter** — writes chapter prose against a chapter brief. Writes `chapters/`.
5. **voice-scrubber** — runs the 10-pass anti-AI scrub. Edits chapters in place.
6. **line-editor** — sentence-level polish. Edits chapters in place.
7. **continuity-checker** — rebuilds SQLite index, audits facts/timeline/character knowledge. Writes `continuity/`.
8. **foreshadowing-tracker** — owns setup→payoff graph. Writes `foreshadow/`.

Per-chapter order: plotter → drafter → voice-scrubber → line-editor → continuity-checker → foreshadowing-tracker.

## Commands (logical operations the skill performs)

These are not slash commands — they're operations the skill executes when the user asks for them.

### `new <premise>`

Create a new book project. Prompt for genre + length target if not given. Scaffold `books/<slug>/` and write `intent.md`. Default genre word-count bands:

| Genre | Low | Standard | High |
|-------|-----|----------|------|
| Thriller | 70k | 90k | 110k |
| Mystery | 70k | 80k | 100k |
| Romance | 60k | 80k | 100k |
| Epic Fantasy | 100k | 140k | 200k |
| Sci-Fi | 80k | 100k | 130k |
| Literary | 60k | 80k | 100k |
| Horror | 70k | 90k | 110k |
| YA | 55k | 70k | 90k |

### `outline`

Run world-builder → character-designer → plotter in sequence. Produces a complete pre-draft package: world, characters, beat sheet, chapter list, setup-payoff graph. Stops here for user review before drafting begins.

### `draft <chapter-number>` or `draft next`

Produce a chapter brief from the plotter's outputs + current continuity state. Run drafter → voice-scrubber → line-editor → continuity-checker → foreshadowing-tracker. If continuity-checker reports `fail`, loop back to drafter with the report and re-run.

### `continuity-pass`

Run a full-book continuity audit. Useful after major edits or before finalization.

### `revise-to-target [<word-count>]`

Two-stage revision (per design decision 8):
1. Read every chapter file directly from `books/<slug>/chapters/chapter-*.md` (in order). The manuscript stitch from `export` is a derived artifact and may not exist yet — operate on the canonical chapter files. Identify thin sections (under-developed scenes, rushed emotional beats) and bloated sections (digressions, overlong description) chapter by chapter.
2. Hand-craft expand/tighten directives per chapter. Re-run the drafter on affected chapters; re-run scrub/line-edit/continuity on each revised chapter.
3. Stop when total word count (summed across chapters) is within ±5% of target AND every chapter still passes continuity.

### `export`

Stitch `chapters/*.md` into `manuscript.md` (insert chapter titles, drop HTML-comment footers). Run `pandoc manuscript.md -o manuscript.pdf` using pandoc's default LaTeX template. A custom fiction-typeset template is optional — if the user has placed one at `templates/pandoc/fiction.latex` (not provided in the scaffold), pass it with `--template templates/pandoc/fiction.latex`; otherwise use the default.

## Continuity contract (read this every session)

The story must hold together from page 1 to the final line. Mechanisms:

1. **Markdown is canonical** for series state. SQLite is a derived index, rebuilt by `tools/build_continuity_index.py` on demand. The continuity-checker runs the rebuild before each audit.
2. **The plotter produces an explicit setup→payoff graph** before drafting. The foreshadowing-tracker owns it thereafter.
3. **The drafter receives a per-chapter brief** stating: beats to advance, setups to plant, payoffs to fire, knowledge state of POV character at start, what they learn, what they still don't know at end.
4. **Continuity-checker runs per chapter AND once full-book before finalization.** Chapters do not advance if continuity status is `fail`.
5. **Character knowledge sections** (`series-bible/characters/<name>.md` — "Knowledge" block) are owned by the continuity-checker after drafting starts. Drafter reads, never writes.

## Voice strategy (per design decision 5)

Four mechanisms stacked:

1. **Anti-AI tells library + scrub pass.** `library/anti-ai/tells-catalog.md` is loaded by drafter and voice-scrubber on every chapter. Scrub-rules apply 10 passes after drafting.
2. **Author-voice imitation.** Optional. If `intent.md` specifies a voice (e.g. `voice: cormac-mccarthy`), drafter loads that profile.
3. **Genre-baseline voice.** Each genre file specifies default voice/tone (chapter length, sentence rhythm, narrator stance).
4. **Read-aloud cadence.** Scrub pass 10. Sentence-length distribution must be irregular.

## How to start

If the user describes a premise:
1. Ask for any missing inputs (genre, length target, standalone vs. series book 1, POV plan).
2. Create `books/<slug>/` and write `intent.md`.
3. Run the `outline` flow (world → characters → plot). Stop for user review.
4. On user go-ahead, run `draft next` repeatedly until the book is complete.
5. Run `revise-to-target`.
6. Run `export`.

If the user has an existing project, read its `intent.md` and `plot/chapter-list.md` to find the current draft state, then resume from the next undrafted chapter.
